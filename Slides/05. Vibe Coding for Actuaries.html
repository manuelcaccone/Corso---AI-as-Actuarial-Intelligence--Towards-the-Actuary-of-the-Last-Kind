🤖 Strumenti AI per R e RStudio

Guida completa per integrare l'Intelligenza Artificiale nel tuo workflow R

🚀 Benvenuto nel Futuro della Programmazione R

Nel mondo della scienza dei dati e della programmazione in R, l'integrazione dell'Intelligenza Artificiale (AI) e dei Large Language Models (LLM) può migliorare significativamente la produttività e l'efficienza.

I Quattro Pilastri dell'AI in R

📦 ellmer

L'interfaccia principale per LLM in R. Supporta molteplici fornitori e funzionalità avanzate.

👨‍💻 GitHub Copilot

AI pair programmer integrato in RStudio per suggerimenti di codice in tempo reale.

🎯 gander

Chat ad alte prestazioni con consapevolezza dell'ambiente R per interazioni contestuali.

⚡ groqR

Accesso ultra-veloce ai modelli LLM tramite l'infrastruttura Groq per inferenza rapida.

💡 Obiettivo del Tutorial: Al termine di questa guida, saprai come configurare e utilizzare tutti e quattro gli strumenti per massimizzare la tua produttività in R.

📦 ellmer: L'Interfaccia Principale per LLM

Step 1: Installazione

▼

Iniziamo installando ellmer dal repository CRAN:

# Installazione del pacchetto ellmer
install.packages("ellmer")

# Caricamento della libreria
library(ellmer)
✅ Verifica: Dopo l'installazione, verifica che il pacchetto sia caricato correttamente.

Step 2: Scelta del Fornitore di Modelli

▼

ellmer supporta una vasta gamma di fornitori. Ecco le opzioni principali:

Anthropic Claude

# Configurazione per Anthropic Claude
chat_anthropic()

OpenAI GPT

# Configurazione per OpenAI GPT
chat_openai()

Google Gemini

# Configurazione per Google Gemini
chat_google_gemini()

Ollama (Locale)

# Configurazione per Ollama (modelli locali)
chat_ollama()
💡 Raccomandazioni:
  • Per principianti: chat_openai() o chat_anthropic()
  • Per uso gratuito: chat_google_gemini()
  • Per privacy: chat_ollama() (modelli locali)

Step 3: Configurazione dell'Autenticazione

▼

La maggior parte dei fornitori richiede una chiave API. Ecco come configurarla:

# Metodo 1: Variabile d'ambiente (RACCOMANDATO)
# Aggiungi al tuo .Renviron:

# OPENAI_API_KEY=tua_chiave_qui
# ANTHROPIC_API_KEY=tua_chiave_qui

# Metodo 2: Durante la sessione
Sys.setenv(OPENAI_API_KEY = "tua_chiave_qui")
⚠️ Sicurezza: Non includere mai le chiavi API direttamente nel codice. Usa sempre variabili d'ambiente.

Step 4: Primo Utilizzo

▼

Creiamo il nostro primo oggetto chat:

# Caricamento della libreria ellmer
library(ellmer)

# Creazione di un oggetto chat
chat <- chat_openai(
  model = "gpt-4o-mini",
  system_prompt = "Sei un assistente R esperto e conciso."
)

# Prima domanda al modello
risposta <- chat$chat("Come creo un ggplot base in R?")

# Visualizzazione della risposta
print(risposta)

👨‍💻 GitHub Copilot: Il Tuo AI Pair Programmer

Step 1: Prerequisiti

▼

Prima di iniziare, assicurati di avere:

  • ✅ Account GitHub attivo
  • ✅ Sottoscrizione Copilot (gratuita per studenti/docenti)
  • ✅ RStudio Desktop 2023.09.0 o successiva
  • ✅ Connessione Internet stabile
💡 Per Studenti: GitHub Copilot è gratuito tramite il programma GitHub Education. Richiedi l'accesso su education.github.com

Step 2: Configurazione in RStudio

▼

Segui questi passaggi per abilitare Copilot:

  1. Apri RStudio
  2. Naviga su Tools → Global Options → Copilot
  3. Spunta "Enable GitHub Copilot"
  4. Clicca "Download and Install" per i componenti Agent
  5. Clicca "Sign In"
  6. Copia il codice di verifica mostrato
  7. Vai su github.com/login/device
  8. Incolla il codice e autorizza le permessi
✅ Verifica: Dovresti vedere un'icona Copilot nella barra di stato di RStudio.

Step 3: Primo Utilizzo

▼

Iniziamo con un esempio pratico:

# Inizia a digitare un commento descrittivo
# Crea un grafico a dispersione di mpg vs wt dal dataset mtcars

# Copilot suggerirà automaticamente:
library(ggplot2)

ggplot(mtcars, aes(x = wt, y = mpg)) +
  geom_point() +
  labs(
    title = "Relazione tra Peso e Consumi",
    x = "Peso (1000 lbs)",
    y = "Miglia per Gallone"
  )
💡 Come Funziona:
  • Copilot analizza il tuo commento
  • Genera suggerimenti come "ghost text" grigio
  • Premi Tab per accettare
  • Premi Esc per rifiutare

Step 4: Tecniche Avanzate

▼

Massimizza l'efficacia di Copilot con questi suggerimenti:

▼

Per ambienti aziendali o configurazioni specifiche:

🎯 gander: Chat Consapevole dell'Ambiente R

Step 1: Installazione

▼

Installa gander dal CRAN o dalla versione di sviluppo:

# Versione CRAN (stabile)
install.packages("gander")

# Versione di sviluppo (più recente)
pak::pak("simonpcouch/gander")

# Caricamento della libreria
library(gander)

Step 2: Configurazione con ellmer

▼

gander funziona insieme a ellmer. Configura il modello che preferisci:

# Configura il modello (esempio con Claude)
options(.gander_chat = ellmer::chat_anthropic())

# Oppure con OpenAI
options(.gander_chat = ellmer::chat_openai(model = "gpt-4o"))

# Per rendere persistente, aggiungi al tuo .Rprofile:
usethis::edit_r_profile()

# Poi incolla: 
options(.gander_chat = ellmer::chat_anthropic())
💡 Scelta del Modello: Claude Sonnet è particolarmente efficace per la scrittura di codice R, mentre GPT-4o offre ottime prestazioni generali.

Step 3: Configurazione Scorciatoia da Tastiera

▼

Per un accesso rapido, configura una scorciatoia:

🖥️ In RStudio:

  1. Vai su Tools → Modify Keyboard Shortcuts
  2. Cerca "gander"
  3. Assegna Ctrl+Alt+G (Windows/Linux) o Ctrl+Cmd+G (macOS)

💜 In Positron:

// Apri Command Palette → "Open Keyboard Shortcuts (JSON)"
// Aggiungi questa configurazione:

{
  "key": "ctrl+cmd+g",
  "command": "workbench.action.extensions.runCommand",
  "args": {
    "command": "gander.chat"
  }
}

Step 5: Funzionalità Avanzate

▼

gander offre funzionalità uniche grazie alla sua consapevolezza dell'ambiente:

🧠 Contesto Automatico

gander rileva automaticamente:

  • Oggetti nell'ambiente: Nomi e struttura di data.frame, liste, funzioni
  • Contenuto dei file: Codice da altri file del progetto
  • Nomi delle colonne: Tipi di dati e struttura dei dataset
  • Immagini e plot: Grafici già creati nella sessione
⚠️ Limitazioni Attuali: gander funziona meglio con file .R. Per file .Qmd e .Rmd, la formattazione delle risposte potrebbe non essere ottimale.
📚 Approfondimenti: Per maggiori dettagli, consulta la vignette con vignette("gander", package = "gander")

⚡ groqR: Inferenza Ultra-Veloce per LLM

Step 1: Cos'è groqR e Perché Usarlo

▼

groqR è un pacchetto R che fornisce accesso all'infrastruttura Groq, specializzata nell'inferenza ultra-veloce di Large Language Models.

🚀 Vantaggi di Groq:

  • Velocità Estrema: Inferenza fino a 10x più veloce rispetto ad altri provider
  • Modelli Potenti: Accesso a Llama, Mixtral, Gemma e altri modelli open-source
  • Costi Competitivi: Prezzi vantaggiosi per l'inferenza rapida
  • Semplicità: API semplice e diretta
💡 Ideale per: Prototipazione rapida, test iterativi, applicazioni che richiedono risposte immediate, e sviluppo di chatbot.

Step 2: Installazione

▼

groqR non è ancora su CRAN, quindi installiamo dalla versione di sviluppo:

# Installazione da GitHub
if (!require(devtools)) install.packages("devtools")
devtools::install_github("socialresearchcentre/groqR")

# Caricamento della libreria
library(groqR)
⚠️ Prerequisito: Dovrai creare un account gratuito su console.groq.com per ottenere la tua API key.

Step 3: Configurazione API Key

▼

Configura la tua chiave API per accedere ai servizi Groq:

# Metodo 1: Variabile d'ambiente (RACCOMANDATO)
# Aggiungi al tuo .Renviron:

# GROQ_API_KEY=tua_chiave_groq_qui

# Metodo 2: Durante la sessione
Sys.setenv(GROQ_API_KEY = "tua_chiave_groq_qui")

# Verifica configurazione
groq_api_key_set()
🔑 Come ottenere la chiave:
  1. Vai su console.groq.com
  2. Crea un account gratuito
  3. Naviga su "API Keys"
  4. Clicca "Create API Key"
  5. Copia la chiave generata

▼

Ecco una configurazione completa da aggiungere al tuo .Rprofile:

# === CONFIGURAZIONE COMPLETA AI TOOLS ===

# Caricamento automatico dei pacchetti
if (interactive()) {
  suppressMessages({
    library(ellmer)
    library(gander)
    
    # Prova a caricare groqR se disponibile
    if (require(groqR, quietly = TRUE)) {
      groq_available <- TRUE
    } else {
      groq_available <- FALSE
    }
  })
  
  # Configurazione ellmer/gander
  options(
    .gander_chat = ellmer::chat_anthropic(
      model = "claude-3-5-sonnet-20241022",
      system_prompt = paste(
        "Sei un assistente R esperto. Fornisci codice pulito,",
        "ben commentato e seguendo le best practice di tidyverse.",
        "Spiega brevemente cosa fa il codice quando necessario."
      )
    )
  )
  
  # Messaggio di benvenuto
  cat("🤖 AI Tools caricati:", 
      "\n  • ellmer: chat LLM completi", 
      "\n  • gander: chat contestuale (Ctrl+Alt+G)",
      "\n  • Copilot: disponibile in RStudio")
  
  if (groq_available) {
    cat("\n  • groqR: inferenza ultra-veloce")
  }
  
  cat("\n\n")
}

# Funzioni helper
quick_chat <- function(prompt) {
  if (!exists(".quick_chat_obj")) {
    .quick_chat_obj <<- ellmer::chat_anthropic()
  }
  .quick_chat_obj$chat(prompt)
}

# Funzione per chat veloce con Groq (se disponibile)
quick_groq <- function(prompt, model = "llama-3.1-8b-instant") {
  if (exists("groq_chat")) {
    groq_chat(prompt = prompt, model = model)
  } else {
    message("groqR non è disponibile. Usa quick_chat() invece.")
  }
}

# Funzione per confronto velocità
speed_test <- function(prompt) {
  cat("🚀 Test di velocità per:", prompt, "\n\n")
  
  # Test Groq
  if (exists("groq_chat")) {
    start_time <- Sys.time()
    groq_result <- groq_chat(prompt, model = "llama-3.1-8b-instant")
    groq_time <- as.numeric(Sys.time() - start_time)
    cat("⚡ Groq:", groq_time, "secondi\n")
  }
  
  # Test ellmer
  start_time <- Sys.time()
  ellmer_result <- quick_chat(prompt)
  ellmer_time <- as.numeric(Sys.time() - start_time)
  cat("🧠 ellmer:", ellmer_time, "secondi\n\n")
  
  return(list(groq = groq_result, ellmer = ellmer_result))
}
